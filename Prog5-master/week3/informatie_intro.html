Introductie WPF

WPF is een afkorting voor Windows Presentation Foundation. Dit Framework biedt je de tools om een rijke user interface te bouwen. Dit is niet de enige unieke eigenschap van WPF. De mogelijkheid tot het data-binden van je view aan je model geeft je een beter scheiding tussen je GUI en je bedrijfslogica.

Naast de onderliggende structuur, die je helpt een nette en onderhoudbare interface te bouwen, ondersteunt WPF ook meerdere mediaformaten. Een van deze formaten is XAML, een markup language die gebaseerd is op XML*. Maar denk hierbij ook aan 2-D en 3-D Graphics, animaties, losse stylesheets, templates, documenten en media files (mp3, mp4).

*XML (Extensible Markup Language) Een markup language die regels en encoding voor een document vastlegt, net als HTML.

WPF is resolutieonafhankelijk. Dit houdt in dat op verschillende resoluties hetzelfde resultaat getoond zal worden. De XAML interfaces worden vector-based gerenderd.

Interface design

WPF biedt je de mogelijkheid om je interfaces op 2 manieren te ontwikkelen. Het definiëren van je GUI in XAML of het opbouwen van je GUI in code ('code-behind').

XAML

<Window x:Class="XAMLCodeBehind.MainWindow"
	xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
	xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
	Title="MainWindow" Height="350" Width="525">
	
	<Grid>
		<Button>Click Me</Button>
	</Grid>
</Window>

Code Behind

public partial class MainWindow : Window
{
	public MainWindow()
	{
		InitializeComponent();

		Button button = new Button();
		button.Content = "Click me!";
		Griddy.Children.Add(button);
	}
}

Het is netjes en onderhoudbaar om het uiterlijk van de applicatie volledig in XAML te definiëren. Het gedrag van de applicatie kan in de 'code-behind' van de applicatie worden gedefinieerd.

Het design van je applicatie heeft op deze manier een losse koppeling met het gedrag van de applicatie. Hierdoor kan een designer los aan de GUI werken terwijl een ontwikkelaar het gedrag van de applicatie tot stand brengt. Er zijn ook verschillende tools om aan de GUI te werken. Een van deze tools is Microsoft Blend.

Je kunt nu vanuit de code-behind file bij de XAML elementen die hebt je gedefinieerd. De waardes die je op je model (bijv. properties) hebt staan kun je nu dus mee geven aan deze elementen. Het is daar en tegen netter om deze informatie niet te 'setten' maar te 'data-binden'.

Data Binding

WPF bied je de mogelijkheid tot het data-binden van informatie aan je view. In de andere variant van .NET (Windows Forms) had je ook al de mogelijkheid tot het binden van je data aan je view. Er werd alleen de mogelijkheid ondersteund tot one-way binding. In WPF kun je nu ook two-way bindingen implementeren. Dit zorgt er voor dat je model altijd up-2-date is van de laatste aanpassingen door de gebruiker. Hierdoor word je model je 'single-source-of-truth' en hoef je je niet af te vragen of het model al is ge√ºpdate door de view.

Introductie XAML

XAML is een afkorting voor XML based Markup language. Even voor de duidelijkheid, je spreek het uit als 'zemmel'. Het is een onderdeel van het WPF platform. Je kunt XAML definiëren in een .XAML file net als in een .HTML file. Elke XAML file heeft meestal een bijbehorende .CS file waar de 'code-behind' staat, maar dit is niet verplicht. De onderstaande technieken ondersteunen XAML.

  • Windows Store apps
  • Windows Phone apps
  • Silverlight
  • WPF

Wij gaan het voornamelijk inzetten voor WPF.

De opbouw van een XAML document is vergelijkbaar met een HTML document. Je bouwt een boom van Elementen en de eigenschappen van deze elementen plaats je in Attributen. Deze geneste objecten leiden tot een 'element tree' die gecompileerd wordt tot een user interface. Je kunt de volgende elementen met XAML definiëren:

  • Windows
  • dialog boxes
  • pages
  • user controls!
  • controls
  • Shapes
  • graphics
<Window
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    Title="Window with Button"
    Width="250" Height="100">

    <!-- Add button to window -->
    <Button Name="button">Click Me!</Button>

</Window>

WPF wordt voornamelijk gebruikt voor desktop-applicaties en Windows Apps. Daarnaast heb je ook nog browser-hosted-applicaties. Deze applicaties hebben de naam XBAP'S. Wij gaan voornamelijk aan de slag met de desktop applicaties.

Je kunt een volledige applicatie opbouwen met verschillende windows en pages. Je kunt zoveel elementen toevoegen als je zelf wilt. Je bent daarentegen wel verplicht tot het gebruiken van een Application element.

In een console-applicatie heb je een Program class met een Main functie. Dit is de plek waar je applicatie begint. Een dergelijke class heb je ook in elementen met XAML. Dit is de Application class en heeft een bijbehorend XAML element wat hem tot stand brengt. In dit element kun je aangeven bij welk window je wilt beginnen.

class Program

    }
}

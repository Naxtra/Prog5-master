*IDE - Integrated development environment

Introductie .NET Framework.
De wereld van .NET verschilt nogal met die van Java. Om te beginnen is .NET platform afhankelijk. Applicaties geschreven in C# werken alleen op Windows machines. Daarnaast wordt er binnen .NET veelvuldig gebruik gemaakt van assemblies. Dit zijn pakketjes met klasses waar jij als ontwikkelaar gebruik van kan maken.

Elke Windows machine heeft een standaard set aan assemblies waar je als ontwikkelaar meteen bij kan. Daarnaast zijn er eindeloos veel assemblies die je kan "include-en" in je project. Verderop deze pagina gaan we meer aandacht besteden aan het onderwerp assemblies.

Hieronder zie je een overzicht van de onderdelen van het Java framework ten opzichte van het .net framework. Kijk eens goed naar de verschillen.

Maak je geen zorgen als een aantal termen nog onbekend voorkomen. We gaan alle onderdelen de komende weken behandelen. Er zijn een aantal belangrijke verschillen.

  • Java is platform onafhankelijk, .Net draait alleen op windows.
  • Java maakt gebruik van 1 grote JDK (Java Development Kit), .NET heeft verschillende kleinere assemblies.
  • Java heeft een Virtual Machine, .NET gebruikt CLR (Common Language Runtime).

Assemblies

Vanuit Java ben je als het goed is al gewend om gebruik te maken van namespaces. Dit is een soort van 'achternaam' van een klasse. Het kan zo maar zijn dat een naam van een klasse 2x voorkomt in een applicatie. Namespaces zorgen er voor dat deze klasse nog steeds uniek zijn.

In Java kun je gebruik maken van een namespace via het Import statement. Binnen .NET doe je dit met het Using statement.

In .NET gaan we nog een laagje dieper. Klasses worden opgedeeld in namespaces, en namespaces zijn weer onderdeel van een library of assembly. In java krijg je een grote JDK (Java Development Kit). Hierin zitten heel veel klasses waar je gebruik van kan maken. Dit is niet het geval in .NET. Hier worden kleinere sets van klasses opgeleverd in assemblies. Deze assemblies vind je terug in je project in de vorm van DLL's. (Dynamic Link Library).


In elk project in .NET kun je referenties (references) leggen naar andere libraries. Deze libraries bevatten namespaces, en deze namespaces bevatten klasses. Elke template die je start binnen Visual Studio bevat een andere set aan standaard assemblies. Naast deze assemblies kun je zelf nog meer assemblies toevoegen. Deze assemblies kunnen zich al op je computer bevinden omdat ze bijvoorbeeld onderdeel zijn van het .NET framework, of komen online van een externe bron.

Bekijk het onderstaande fimpje voor meer uitleg over assemblies in .NET.
